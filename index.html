📡 SRT 中繼伺服器使用教學

給借用伺服器台主的完整設定指南（新手適用）

整體架構
📱 你的手機 IRL Pro
→
🖥️ 中繼伺服器 flycat.ddns.net
→
🎬 你的 OBS 拉流 + 推 Twitch
→
🟣 Twitch 你的頻道
白話說：手機畫面先傳到中繼伺服器存著，你的電腦 OBS 再從伺服器抓畫面，然後推到 Twitch。這樣手機訊號不穩也不會直接閃退直播。
1 你的串流資訊
以下資訊由伺服器管理員提供，每位台主不同，請勿分享給他人。
項目你的設定值
推流網址srt://flycat.ddns.net:5002
Stream ID（推流用）publish:你的名字
OBS 拉流網址srt://flycat.ddns.net:5002?streamid=read:你的名字
NOALBS 監測網址http://flycat.ddns.net:9997/v3/paths/get/你的名字
注意：「你的名字」由管理員指定，請直接替換整個詞，例如 publish:kevin
2 手機 IRL Pro 設定

下載 IRL Pro

Android 手機到 Google Play 搜尋「IRL Pro」安裝（免費）。

新增串流設定

打開 IRL Pro → 右上角齒輪 → Stream → 加號新增 → 選擇 SRT

填入以下資料

Host（主機位址）
flycat.ddns.net
Port（連接埠）
5002
Stream ID
publish:你的名字

建議畫質設定

項目建議值
解析度1080p 或 720p
Bitrate3000–5000 kbps（看網路狀況）
關鍵影格間隔2 秒
3 OBS 設定

新增媒體來源（拉流）

OBS → 來源 → 加號 → 媒體來源 → 取消勾選「本機檔案」

輸入以下網址
srt://flycat.ddns.net:5002?streamid=read:你的名字

勾選「重新連接時循環」，其他保持預設。

設定推流到你的 Twitch

OBS → 設定 → 串流 → 選擇 Twitch → 連結帳號或手動貼上串流金鑰。

建立三個場景

IRL
正常直播畫面
媒體來源放這裡
lowB
訊號差時自動切換
可放等待圖片
BRB
斷線時自動切換
離開畫面用
場景名稱要一模一樣，NOALBS 才認得到。
4 NOALBS 設定（自動場景切換）

下載 NOALBS

前往 https://github.com/NOALBS/nginx-obs-automatic-low-bitrate-switching/releases
下載 noalbs-v2.x.x-x86_64-pc-windows-msvc.zip，解壓縮到任意資料夾。

建立 .env 檔案

在 NOALBS 資料夾裡新增一個檔案，命名為 .env（注意前面有點），填入：

TWITCH_BOT_USERNAME=你的BOT帳號
TWITCH_BOT_OAUTH=oauth:你的token
沒有 BOT 帳號？用自己的 Twitch 帳號也可以，OAUTH 到 https://twitchapps.com/tmi/ 取得。

建立 config.json 設定檔

在同一個資料夾建立 config.json，貼上以下內容，並把 三個地方 換成你的資料：

{ "user": { "id": null, "name": "你的Twitch頻道名稱", "passwordHash": null }, "switcher": { "bitrateSwitcherEnabled": true, "onlySwitchWhenStreaming": true, "instantlySwitchOnRecover": true, "autoSwitchNotification": true, "retryAttempts": 5, "triggers": { "low": 800, "rtt": 2500, "offline": 100 }, "switchingScenes": { "normal": "IRL", "low": "lowB", "offline": "BRB" }, "streamServers": [ { "streamServer": { "type": "Mediamtx", "statsUrl": "http://flycat.ddns.net:9997/v3/paths/get/你的名字" }, "name": "MediaMTX", "priority": 1, "overrideScenes": null, "dependsOn": null, "enabled": true } ] }, "software": { "type": "Obs", "host": "localhost", "password": "你的OBS WebSocket密碼", "port": 4455 }, "chat": { "platform": "Twitch", "username": "你的BOT帳號", "admins": ["你的Twitch頻道名稱"], "ignoreUsers": [], "language": "EN", "prefix": "!", "enablePublicCommands": false, "enableModCommands": true, "enableAutoStopStreamOnHostOrRaid": false, "announceRaidOnAutoStop": false, "commands": { "Fix": { "permission": "Mod", "userPermissions": null, "alias": ["f"] }, "Switch": { "permission": "Mod", "userPermissions": null, "alias": ["ss"] }, "Bitrate": { "permission": null, "userPermissions": null, "alias": ["b"] }, "Start": { "permission": "Mod", "userPermissions": null, "alias": ["start"] }, "Stop": { "permission": "Mod", "userPermissions": null, "alias": ["stop"] } } }, "optionalScenes": { "starting": null, "ending": null, "privacy": null, "refresh": null }, "optionalOptions": { "twitchTranscodingCheck": false, "twitchTranscodingRetries": 5, "twitchTranscodingDelaySeconds": 15, "offlineTimeout": null, "recordWhileStreaming": false, "switchToStartingSceneOnStreamStart": false, "switchFromStartingSceneToLiveScene": false } }

開啟 OBS WebSocket

OBS → 工具 → WebSocket 伺服器設定 → 啟用 → 設定密碼 → 確定。
把密碼填到 config.json 的 "password" 欄位。

啟動 NOALBS

執行 noalbs.exe，看到沒有錯誤訊息就代表成功連上。

5 聊天室指令
指令功能誰能用
!start開始直播台主 / 管理員
!stop停止直播台主 / 管理員
!ss IRL手動切換到正常場景台主 / 管理員
!ss BRB手動切換到等待場景台主 / 管理員
!b查詢目前訊號強度所有人
6 每次直播前的流程

① 開啟 OBS

確認媒體來源已設好，三個場景都在。

② 執行 noalbs.exe

等它連上 OBS（顯示 Connected to OBS）。

③ 手機開 IRL Pro 開始推流

確認手機有網路，按下開始推流。

④ 在聊天室打 !start

NOALBS 收到指令後會自動開始直播。

自動場景切換邏輯
狀況切換到觸發條件
訊號正常IRL 場景bitrate > 800 kbps
訊號變差lowB 場景bitrate 介於 100–800 kbps
訊號斷線BRB 場景bitrate < 100 kbps
訊號恢復後會自動切回 IRL，不需要手動操作。
