📡 戶外直播設定教學

你應該已收到一個壓縮包（內含 install_noalbs.bat）和一份直播連線資料文字檔

這套系統在幹嘛？

讓手機畫面先傳到中繼站，再從中繼站傳到你的電腦 OBS。這樣手機訊號短暫斷掉，直播也不會馬上中斷。

📱你的手機IRL Pro / Moblin
→
🖥️中繼站幫你暫存畫面
→
🎬你的電腦 OBS抓畫面 + 推直播
→
🟣Twitch你的頻道
1 跑安裝工具（只要做一次）

解壓縮收到的壓縮包，雙擊 install_noalbs.bat，依照提示輸入五項資料：

工具詢問填什麼
① Twitch ID你的 Twitch 英文帳號
② Bot TokenNOALBS 會用你自己的帳號在聊天室發通知，不需要另外建機器人帳號。工具會幫你開網頁，確認已用你的帳號登入 Twitch 後，按 Connect，複製 oauth:xxxxxxxxxx 貼回來
③ 伺服器選擇輸入 1（flycat 伺服器）
④ OBS WebSocket 密碼OBS → 工具 → WebSocket 伺服器設定 裡自己設的密碼（見 Step 3）
⑤ 場景名稱直接按 Enter 使用預設（IRL / lowB / BRB）
完成後資料夾裡會有 啟動_NOALBS.bat 和 關閉_NOALBS.bat，之後每次直播用這兩個開關 NOALBS。
2 手機設定
URL 和 Stream ID 請整串貼上收到的直播連線資料文字檔裡的內容，不要自己修改。少了推流帳號或密鑰的部分（例如只填 publish:你的TwitchID）會連不上中繼站。
A

Android — IRL Pro（Google Play 免費下載）

① 進入推流設定

打開 IRL Pro，點左上角的齒輪圖示 ⚙ → 選 Connections（連線設定）→ 點右上角 New connection。

② 選擇協定

出現協定選擇時，選 SRT。

③ 填入連線資料

App 欄位名稱填什麼
Name（設定名稱）隨便取，例如 flycat
URL直播連線資料文字檔裡的「URL / Host」那行，整串貼上
streamid直播連線資料文字檔裡的「Stream ID」那行，整串貼上
latency（延遲）填 2500
Mode選 Video + Audio
如果 App 版本沒有 URL 欄位，改填：Host 填文字檔裡的域名部分，Port 填 5002，streamid 貼整串。

④ 畫質設定

在設定頁面找到 Video 區塊：Resolution 選 1920×1080、FPS 選 30、Bitrate 填 4000、Keyframe frequency 填 2。

⑤ 啟用這組設定並開始推流

回到 Connections 清單，點你剛建的設定前面的圓圈讓它變成選取狀態 → 回主畫面按 Go Live。看到綠色連線狀態就代表成功傳到中繼站了。

出現黑畫面：把手機 App 的影像編碼改成 H264 / AVC（H265 / HEVC 部分電腦 OBS 無法解碼）。
B

iPhone — Moblin（App Store 免費下載）

① 進入串流設定

打開 Moblin，點畫面上的齒輪 ⚙️ → 選 Streams（串流設定）→ 點右上角 ＋ 新增一組。

② 填入連線資料

App 欄位名稱填什麼
Name（設定名稱）隨便取，例如 flycat
Protocol（協定）選 SRT(LA)
URL直播連線資料文字檔裡的「URL / Host」那行，整串貼上
Stream ID直播連線資料文字檔裡的「Stream ID」那行，整串貼上
Latency（延遲）填 2500

③ 畫質設定

⚙️ → Camera → Video：Resolution 選 1080p、FPS 選 30、Bitrate 填 4000 kbps。

④ 開啟背景串流

⚙️ → Streams → 選你剛建的設定 → 往下滑 → 開啟 Background streaming（這樣螢幕關掉或切 App 時推流不會中斷）。

⑤ 開始推流

回主畫面，點下方推流按鈕開始推流。畫面出現連線資訊代表成功。

出現黑畫面：把影像編碼改成 H264 / AVC。
3 電腦 OBS 設定
沒有 OBS 的話，先到 obsproject.com 下載安裝免費版。
1

建立三個場景

場景就像是不同的「畫面模板」，訊號好壞時會自動切換。在 OBS 左下角「場景」欄位，按 ＋ 新增三個，名稱要完全一樣（包含大小寫）：

IRL
訊號正常時的主畫面
lowB
訊號變差時自動切換
BRB
完全斷線時自動切換

lowB 和 BRB 場景可以放靜態圖片或文字，讓觀眾知道你暫時斷線；IRL 場景放手機畫面（下一步設定）。

2

在 IRL 場景加入手機畫面

先點選左下角的 IRL 場景，然後在右下角「來源」欄位按 ＋ → 選「媒體來源」→ 取消勾選「本機檔案」。

在「輸入」欄位貼上直播連線資料文字檔裡的 OBS 媒體來源那一整串網址。

然後設定以下兩個欄位：

欄位填什麼
輸入格式mpegts
重新連線延遲3（秒）

設好後按「確定」。OBS 畫面應該會出現手機傳來的畫面（手機要先開始推流）。

OBS 網址也要整串貼上。少了拉流帳號或密鑰的部分會連不上中繼站，畫面會是黑的。
有聲音但黑畫面：把手機 App 的影像編碼改成 H264 / AVC，H265 / HEVC 部分電腦 OBS 無法解碼。
3

設定推流到 Twitch

OBS 上方選單 → 設定 → 串流 → 服務選 Twitch → 點「連結帳號」→ 用瀏覽器登入你的 Twitch 帳號 → 授權。

4

開啟 WebSocket（讓 NOALBS 可以控制 OBS）

OBS 上方選單 → 工具 → WebSocket 伺服器設定 → 勾選「啟用 WebSocket 伺服器」→ 設定一組密碼（自己記住，Step 1 安裝工具會問你這個密碼）→ 按「確定」。

如果你已經先跑完 Step 1 的安裝工具，記得確認這裡的密碼和你當時填的一樣。
4 聊天室指令
指令效果誰能用
!start開始直播台主 / 管理員
!stop停止直播台主 / 管理員
!ss IRL手動切回正常畫面台主 / 管理員
!ss BRB手動切到等待畫面台主 / 管理員
!b查詢目前訊號強度所有觀眾
5 每次直播前的順序
1

開電腦 OBS

確認三個場景都在，IRL 場景有媒體來源。

2

雙擊「啟動_NOALBS.bat」

NOALBS 在背景執行，不會有視窗。稍等幾秒後在聊天室打 !b，有回應就代表連線成功。

3

手機開始推流

IRL Pro 或 Moblin 按推流按鈕，畫面開始傳到中繼站。

4

聊天室打 !start

NOALBS 自動幫你開始直播。直播結束後雙擊「關閉_NOALBS.bat」關閉。
