📡 戶外直播設定教學

主要照設定工具做；這頁是看不懂欄位時的輔助說明

管理員需要給台主什麼？

台主不用自己猜網址或密鑰。管理員用白名單工具新增台主後，把產生的資料交給台主即可。

要給台主的東西用途
直播連線資料文字檔裡面有手機 URL、手機 Stream ID、OBS 媒體來源網址
install_noalbs.bat台主第一次設定 NOALBS 時雙擊執行；NOALBS 資料夾會建立在這個 bat 旁邊
這個教學網頁台主忘記欄位怎麼填時看這頁
直播連線資料文字檔在哪裡？管理員端白名單工具會輸出在 projects/irl-stream/config/exports/台主ID-settings.txt。
中繼站如果搬到筆電：台主不用自己改路由器或猜新 IP。管理員會重新確認伺服器，必要時重新給一份新的直播連線資料文字檔。
不要只丟網址給台主。白名單版一定需要完整 Stream ID 和完整 OBS 媒體來源網址，少了推流 / 拉流密鑰會連不上。
不要傳管理員本機測試網址。127.0.0.1:8890 只有管理員這台電腦能用，外部台主要用 flycatirl.ddnsgeek.com:5002。
這套系統在幹嘛？

戶外直播最怕網路不穩，一閃退整個直播就掛了。
這套方法是讓手機畫面先傳到一台中繼站，再從中繼站傳到你的電腦 OBS，這樣就算手機訊號短暫斷掉，直播也不會馬上中斷。

→
🖥️ 中繼站 幫你暫存畫面
→
🎬 你的電腦 OBS 抓畫面 + 推直播
→
🟣 Twitch 你的頻道
1 先跑一條龍設定工具

主要流程只有一件事：先跑管理員給你的「戶外直播一條龍設定工具」。手機的 Stream ID 和 OBS 的拉流網址請貼管理員給你的完整文字，不要自己手動組。

請先雙擊
install_noalbs.bat

工具會依序問你五件事，照提示貼上即可：

工具詢問填什麼
① Twitch ID你的 Twitch 英文帳號，例如 kevin
② Bot Token工具會幫你開網頁，複製 oauth:xxxxxxxxxx 貼回來
③ 伺服器選擇通常選 1（flycat 伺服器），不確定就問管理員
④ OBS WebSocket 密碼你在 OBS → 工具 → WebSocket 伺服器設定 自己設的密碼
⑤ 場景名稱直接按 Enter 就好（預設 IRL / lowB / BRB）

完成後會自動開啟 NOALBS 資料夾。資料夾會放在 install_noalbs.bat 旁邊，名稱像 NOALBS_你的TwitchID，裡面會有：

檔案用途
.envNOALBS 登入 Twitch 聊天室用
config.jsonNOALBS 控制 OBS 和監測中繼站用
啟動_NOALBS.bat以後每次直播前用它啟動 NOALBS

如果你之後忘記某個資料在哪，可以照這張表找：

想找的資料在哪裡
手機推流 URL / Host / Port管理員給你的直播連線資料
手機 Stream ID管理員給你的直播連線資料，請整串貼上
OBS 媒體來源網址管理員給你的直播連線資料，標題通常會寫「OBS 媒體來源」
OBS 場景名稱config.json 裡的 switchingScenes，預設是 IRL / lowB / BRB
NOALBS 監測中繼站的網址config.json 裡的 statsUrl
Twitch Bot Token.env 裡的 TWITCH_BOT_OAUTH，不要傳給別人
OBS WebSocket 密碼config.json 裡的 software.password，不要公開
最常用的兩個：手機 Stream ID 和 OBS 媒體來源網址都看「管理員給你的直播連線資料」，不要去 .env 或 config.json 裡手動拼。
Token 不要傳給別人。Token 只需要貼進設定工具。手機和 OBS 的欄位看下面章節填。
2 手機安裝 IRL Pro
先跑完第 1 步。然後照這裡填手機推流設定。
1

安裝 App

Android 手機到 Google Play 搜尋「IRL Pro」安裝（免費）。

iPhone 用戶請跳到下方的「iOS 手機設定」章節。IRL Pro 沒有 iOS 版。
2

進入設定頁面

打開 IRL Pro，點左上角的齒輪圖示（⚙）進入設定。

找到 「Connections」（連線設定），點進去。

3

新增推流連線

進入 Connections 頁面後，點 「New connection」 新增一組設定。

出現協定選擇時，選 「SRT」。

4

填入連線資料

依序填入以下欄位：

App 上的欄位名稱填入的內容
Name（設定名稱）隨便取，例如 flycat
URL（伺服器位址）srt://flycatirl.ddnsgeek.com:5002
Mode選 Video + Audio
streamidpublish:你的 Twitch ID:推流帳號:推流密鑰
latency（延遲）2500
如果 App 版本沒有 URL 欄位：Host 填 flycatirl.ddnsgeek.com，Port 填 5002，streamid 貼管理員給你的完整 Stream ID。
streamid 不要自己簡化。白名單版會有推流帳號和密鑰，只填 publish:你的 Twitch ID 會連不上。
5

畫質設定（Video）

設定頁面找到 「Video」 區塊，調整以下欄位：

欄位名稱建議設定說明
Resolution（解析度）1920×1080網路差可改 1280×720
FPS（影格率）30想要更順可以測 60，但手機和網路要夠穩
Bitrate（畫面品質）3000～5000數字越高越清晰，但也越耗網路
Keyframe frequency（關鍵影格）2固定填 2
想測 60 fps：先把 FPS 改 60、Bitrate 提到 5000～8000 kbps。如果 OBS 黑畫面，請把編碼改成 H264 / AVC；若畫面卡頓或常斷線，就退回 30 fps。
6

選用這組設定

回到 Connections 清單，點你剛才新增的設定前面的 圓圈，讓它變成選取狀態，代表推流時使用這組設定。

7

開始推流

回到主畫面，點畫面下方的 「Go Live」 按鈕開始推流。

畫面上出現 綠色連線狀態 代表成功連到中繼站，可以繼續下一步了。

看到 「Connecting...」一直轉？確認 Host 和 Port 有沒有打錯，或詢問管理員伺服器是否正常。
3 iOS 手機安裝 Moblin（iPhone 用）
先跑完第 1 步。然後照這裡填 iPhone 推流設定。
1

安裝 App

到 App Store 搜尋「Moblin」安裝（免費）。

2

進入串流設定

打開 Moblin，點畫面上的 ⚙️ 齒輪 進入設定。

選 「Streams」（串流設定），點右上角 「＋」 新增一組。

3

填入連線資料

依序填入以下欄位：

App 上的欄位名稱填入的內容
Name（設定名稱）隨便取，例如 flycat
Protocol（協定）選 SRT(LA)
URLsrt://flycatirl.ddnsgeek.com:5002
Stream IDpublish:你的 Twitch ID:推流帳號:推流密鑰
Latency（延遲）2500
如果 App 版本把欄位拆開：Host 填 flycatirl.ddnsgeek.com，Port 填 5002，Stream ID 貼管理員給你的完整 Stream ID。
Stream ID 不要自己簡化。白名單版會有推流帳號和密鑰，只填 publish:你的 Twitch ID 會連不上。
URL 開頭填 srt:// 就是純 SRT 模式，與本伺服器相容。不要填 srtla:// 開頭。
4

畫質設定

設定頁面找到 「Camera」→「Video」，調整以下欄位：

欄位名稱建議設定說明
Resolution（解析度）1080p網路差可改 720p
FPS（影格率）30想要更順可以測 60，但手機和網路要夠穩
Bitrate（畫面品質）3000～5000 kbps數字越高越清晰
想測 60 fps：先把 FPS 改 60、Bitrate 提到 5000～8000 kbps。如果 OBS 黑畫面，請改用 H264 / AVC；若畫面卡頓或常斷線，就退回 30 fps。
5

開啟背景串流

⚙️ → Streams → 選你剛建的設定 → 往下滑 → 開啟 「Background streaming」。

這樣手機螢幕關掉或切換 App 時，推流不會中斷。

6

開始推流

回主畫面，點下方 推流按鈕 開始推流。

畫面出現連線資訊代表成功，可以繼續下一步了。

4 電腦 OBS 設定
沒有 OBS 的話，先到 https://obsproject.com 下載安裝。
OBS 要填的拉流網址請貼管理員給你的完整 OBS 媒體來源輸入，不要自己手動組。
管理員本機測試例外：如果 OBS 跟中繼伺服器開在同一台電腦，請用管理員提供的 127.0.0.1:8890 本機網址。一般台主自己的 OBS 才用下面的外部網址。
1

建立三個場景

場景就像是不同的「畫面模板」，訊號好壞時會自動切換。

在 OBS 左下角「場景」欄位，按加號新增三個，名稱要完全一樣：

IRL
訊號正常時
顯示手機畫面
lowB
訊號變差時
自動切換到這
BRB
完全斷線時
自動切換到這
2

把手機畫面加到 IRL 場景

點選 IRL 場景 → 右下角「來源」欄位按加號 → 選「媒體來源」→ 取消勾選「本機檔案」

在「輸入」欄位貼上：

srt://flycatirl.ddnsgeek.com:5002?streamid=read:你的 Twitch ID:拉流帳號:拉流密鑰

「輸入格式」填 mpegts，「重新連線延遲」填 3 秒；建議勾選「來源啟用時重新開始播放」和「播放結束時不顯示任何內容」→ 確定。

OBS 網址也不要自己簡化。白名單版會有拉流帳號和密鑰，只填 read:你的 Twitch ID 會連不上。
有聲音沒畫面或黑畫面：請把手機 App 的影像編碼改成 H264 / AVC。H265 / HEVC 不是每台電腦的 OBS 都能穩定解。
3

設定推流到 Twitch

OBS 上方選單 → 設定 → 串流 → 服務選 Twitch → 連結你的 Twitch 帳號。

4

開啟 WebSocket（讓 NOALBS 能控制 OBS）

OBS 上方選單 → 工具 → WebSocket 伺服器設定 → 勾選「啟用 WebSocket 伺服器」→ 設定一組密碼（自己記住）→ 確定。

這組密碼等等要填到 NOALBS 設定裡，不要忘記。
5 NOALBS 設定（自動切場景的程式）
NOALBS 是什麼？一個在背景跑的小程式，它會一直盯著你的訊號強弱，自動幫你切換 OBS 場景，不用你手動操作。
注意：這一段是在你的直播電腦上做，不是在管理員的伺服器上做。中繼伺服器由管理員負責開，你只要設定自己的 OBS 和 NOALBS。
不用手動設定 NOALBS。第 1 步的一條龍工具會自動下載 NOALBS 並產生設定檔，這裡只是說明工具做了什麼。
1

取得 Twitch Bot Token

NOALBS 需要一個 Twitch 授權碼才能在聊天室發言。

① 打開這個網址（建議用你的帳號登入 Twitch 後再開）：

https://irlhosting.com/tmi/

② 按「Connect」→ 授權 → 複製出現的 oauth:xxxxxxxxxx，這就是你的 Token。

Token 不要貼給別人。只有等等安裝工具會用到。
2

如果還沒跑設定工具，先回到第 1 步

NOALBS 設定會由一條龍工具自動產生，你不用手動下載 NOALBS，也不用自己建立 .env 或 config.json。

3

依序輸入五個資料

第 1 步的工具會問你五件事，照著貼上：

工具詢問你要填什麼
① Twitch ID你的 Twitch 英文帳號，例如 kevin
② Bot Token剛才複製的 oauth:xxxxxxxxxx
③ 伺服器選擇輸入 1 使用 flycat 伺服器（不確定就選這個）
④ OBS WebSocket 密碼你在 OBS WebSocket 設定裡自己設的密碼
⑤ 場景名稱直接按 Enter（預設 IRL / lowB / BRB）

工具會自動下載 NOALBS，並幫你建立 .env、config.json 和啟動檔。資料夾會放在 install_noalbs.bat 旁邊，名稱像 NOALBS_你的TwitchID。

4

執行 NOALBS

安裝完成後，工具會打開 NOALBS 資料夾。之後每次直播前，雙擊：

啟動_NOALBS.bat
NOALBS 資料夾在哪？它會在 install_noalbs.bat 旁邊，名稱像 NOALBS_你的TwitchID。
要關閉 NOALBS？直接關掉 noalbs.exe 的黑色視窗即可；如果看不到視窗，就打開工作管理員，結束 noalbs.exe。
連不上 OBS？先確認 OBS 有開、WebSocket 有啟用、密碼有填對。
監測網址連不上？先問管理員中繼伺服器是否已開，不需要自己修 Docker 或 MediaMTX。
6 聊天室指令

在 你的 Twitch 聊天室 打這些指令來控制直播：

指令效果誰能用
!start開始直播台主 / 管理員
!stop停止直播台主 / 管理員
!ss IRL手動切回正常畫面台主 / 管理員
!ss BRB手動切到等待畫面台主 / 管理員
!b查詢目前訊號強度所有觀眾
7 每次直播前的順序
1

開電腦 OBS

確認三個場景都有，IRL 場景裡有媒體來源。

2

執行啟動_NOALBS.bat

等黑色視窗出現「Connected to OBS」就代表連上你的 OBS 了。

直播結束後，關掉 noalbs.exe 黑色視窗即可停止 NOALBS。

3

手機開 IRL Pro 開始推流

按下推流按鈕，手機畫面會開始傳到中繼站。

4

在聊天室打 !start

NOALBS 收到指令後會自動幫你開始直播，不用在 OBS 手動按。

訊號好壞會自動切換場景

不用你手動操作，NOALBS 會一直偵測訊號，自動切換：

訊號狀況自動切到
訊號正常IRL（正常直播畫面）
訊號變差lowB（低訊號等待畫面）
訊號完全斷掉BRB（離開畫面）
訊號恢復後會自動切回 IRL，不用你操作。
